## Запуск приложения
- Для запуска приложения используется команда:  
  ```bash
  docker-compose up

## 1. Собрано приложение с использованием Python/Django
- Приложение подготовлено на основе легковесного образа `python:3.9-alpine`.
- Все зависимости указаны в `requirements.txt` и устанавливаются без кеширования.

## 2. Докеризация приложения
- Создан `Dockerfile`, который:
  - Использует базовый образ `alpine`.
  - Добавляет утилиты, необходимые для сборки и работы с базой данных PostgreSQL.
  - Настраивает приложение для работы через непривилегированного пользователя.
  - Оптимизирует сборку путем удаления кеша после установки зависимостей.

## 3. Настроен `docker-compose.yml`
- Включает два сервиса:
  - `web` (Django-приложение).
  - `db` (PostgreSQL на базе легковесного образа `postgres:14-alpine`).
- Вся конфигурация осуществляется через переменные окружения.
- Используются внешние тома (`volumes`) для хранения статических данных и базы данных.

## 4. Автоматизация миграций
- При старте контейнера `web` выполняются автоматические миграции базы данных с помощью команды `python manage.py migrate`.

## 5. Учет безопасности
- Контейнер запускается от непривилегированного пользователя `appuser`.
- Минимизирован объем установленного ПО, чтобы снизить возможные риски.

## 6. Легковесность решения
- Использованы легковесные образы `alpine` для Python и PostgreSQL.
- Удален кеш после установки зависимостей.

## 7. Проверено рабочее состояние
- Приложение запускается успешно, автоматически применяет миграции и подключается к базе данных.
- Конфигурация `docker-compose` протестирована: оба контейнера (`web` и `db`) корректно взаимодействуют.
